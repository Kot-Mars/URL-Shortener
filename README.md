# 🔗 URL Shortener Service

Микросервис на Java 17 с использованием Spring Boot для сокращения длинных ссылок. Предназначен для быстрого доступа к URL через короткие хэши, с кешированием, асинхронной генерацией и очисткой старых данных. Полностью готов к развёртыванию в Docker.

---

## 📌 Что решает этот сервис

- Уменьшает длину URL-адресов для удобства использования и шаринга
- Быстро перенаправляет пользователей без задержек за счёт Redis-кеша
- Автоматически освобождает ресурсы, удаляя устаревшие ссылки
- Генерирует хэши заранее — минимальные задержки при создании ссылок
- Предотвращает дублирование: повторный URL вернёт тот же хэш

## 🛠 Запуск через Docker

### 📦 Сборка

```bash
 ./run.sh
```

Скрипт run.sh выполняет:
``` bash  
  docker compose build
docker compose -p url-shortner-container up -d
```
### 🛑 Остановка
``` bash
 ./stop.sh
``` 

### 🔍 Swagger UI

Swagger UI предоставляет удобный веб-интерфейс для тестирования и документации REST API, реализованного в этом сервисе.

#### 📌 Где доступен

##### После запуска приложения Swagger UI будет доступен по адресу:
- [http://localhost:8091/swagger-ui/index.html](http://localhost:8091/swagger-ui/index.html)

#### 📘 Что позволяет Swagger UI

- Просматривать все доступные эндпоинты сервиса
- Отправлять запросы напрямую из браузера
- Смотреть примеры запросов и ответов
- Тестировать поведение API без написания кода
- Проверять валидацию входных данных

#### ✅ Основные эндпоинты

| Метод | URL                             | Описание                                      |
|-------|----------------------------------|-----------------------------------------------|
| POST  | `/api/v1/urls`                  | Создание короткой ссылки                       |
| GET   | `/api/v1/urls/{hash}`           | Перенаправление по короткой ссылке            |

Если интерфейс Swagger не открывается, проверь:

- Приложение действительно запущено на порту `8091`
- Включён ли модуль SpringDoc OpenAPI
- Отображаются ли логи и нет ли ошибок при старте

### 💾 Redis и TTL
#### Redis используется как быстрый кеш:

- Ключ: хэш

- Значение: оригинальный URL

- TTL (spring.data.redis.ttl: 1) — 1 день
⏱️ После чего данные автоматически удаляются из кеша

### 🧠 PostgreSQL
#### Используется как основное хранилище:

- Таблица url хранит соответствия {hash, url, created_at}

- Таблица hash управляет пулом доступных хешей

- Хэши создаются заранее (batch) и кешируются в очереди

### 🧼 Очистка старых ссылок
#### Планировщик CleanerScheduler:

- Удаляет URL, старше 1 года

- Освобождает хэши и возвращает их в пул

#### Запуск по cron:
```
scheduled:
  cleaner:
    cron: "0 0 0 * * *" # каждый день в полночь
```
